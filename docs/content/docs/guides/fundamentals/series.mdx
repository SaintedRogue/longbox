A **Series** is your run of a title — a lot of collectors get mixed up on what that means in Longbox, especially for those with primarily eBook libraries. This guide clears up the confusion.

## What Defines a Series?

<Callout type="idea">
	**PULL LIST** — There are a few open issues that discuss the future expansion of what a valid
	series can be, namely via library support for [nested series](https://github.com/stumpapp/stump/issues/280) and
	[metadata-only series](https://github.com/stumpapp/stump/issues/288). If these are features you
	are interested in, please let us know by thumbing up the issue.
</Callout>

A series is just a collection of items that are related to each other, typically in a linear fashion (e.g., a book series or a specific comic run). In Longbox, a series is defined primarily by its location and structure within a library, since Longbox does not _currently_ support metadata-only library construction.

There are two supported library patterns today:

- **Collection priority library**: Takes the top most folder and collapses everything into it as a single series.
- **Series priority Library**: Takes the bottom most folders as series, effectively being the opposite of a collection based library.

There are plans to expand this in the future, but for now, this is what is supported.

These patterns have more to do with how the library is structured than the actual series itself, so to see additional information on these patterns be sure to visit the [relevant](/docs/guides/fundamentals/libraries) section of the documentation.
